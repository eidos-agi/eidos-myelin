Claude Code has perfect recall and zero learning. Every session starts over. Myelin changes that.

Myelin is the biological sheath that wraps around nerve fibers. It doesn't create new pathways. It makes existing ones faster. Signals that travel myelinated axons move 100x faster than bare ones.

Practice builds myelin. Repetition builds myelin. Experience builds myelin.

Same neurons. Radically faster.

eidos-myelin watches how you use Claude Code — across sessions, across projects, across weeks — and distills what works into something it can use next time. Not instructions you write. Patterns it observes.

Built on claude-session-commons, which already parses every session transcript, chunks it, embeds it, and extracts per-session patterns. Myelin adds the cross-session layer: aggregation, promotion, confidence scoring, and retrieval.

01 Three kinds of learned knowledge

Not all patterns are created equal. Myelin distinguishes between things that work, things that don't, and how you specifically prefer to operate.

Tactic

What consistently works

Cross-session patterns promoted after repeated observation. Confidence-scored. Decay without reinforcement. Scoped to a project or global.

confidence: 0.87 · seen 12x · contradicted 0x

"grep --type yaml finds config in 1 call.
 Don't glob first. Don't read files speculatively."
Anti-pattern

What to stop doing

Distilled from sessions with high tool-call counts and low progress. User corrections. Repeated failures. Research shows these are more valuable than positive patterns.

ANTI-PATTERN: over-reading
confidence: 0.95 · user corrected 3x

"Reading >2 files before writing correlates
 with user frustration in this project."
User Model

How you work best

Behavioral, not biographical. Predicts what makes sessions productive. Transparent — you can read it, edit it, reset it.

Every tactic has a lifecycle: draft → candidate → active → stale → retired. Nothing becomes active without human approval. This is a guardrail, not a bottleneck — the research shows why.

02 How it works

The existing commons daemon already parses and indexes every session transcript in the background. Myelin adds a distillation pass after indexing, and an MCP server that surfaces results at session boot.

# the pipeline

sessions/*.jsonl
  → commons.parse()  # exists
  → commons.chunk()  # exists
  → commons.embed()  # exists
  → commons.analyze_patterns()  # exists — per session

  → commons.distill()  # NEW — cross-session
  → human reviews  # myelin review CLI
  → tactics.db  # in insights.db

# at session boot (<100ms)
myelin.tactics_for_project(path) → top 5

# the loop that makes it self-improving
myelin.tactic_feedback(id, worked=true)
  → confidence ++ | threshold recalibrates
  → meta-tactics about myelin itself emerge

03 What the research says

18 findings. Two empirical studies. Six prior art systems surveyed. One GPT-5.2 peer review. The decision was earned through evidence, not intuition.

Confirmed

Auto-generated context hurts performance

LLM-generated AGENTS.md files reduce task success by 2–3% and increase costs 20%+. Human-written files improve success by 4%. The human gate is the proven differentiator.

ETH Zurich · arxiv 2602.11988 · 138 tasks, 4 agents (Claude Code, Codex, GPT, Qwen)
Confirmed

Nothing does cross-session learning yet

Reflexion reflects within trials. A-Mem links within sessions. Mem0 remembers conversational facts. AGENTS.md persists but requires manual curation. The cross-session gap is real and unfilled.

6 systems surveyed · Reflexion, A-Mem (NeurIPS '25), Mem0, AutoGen, AGENTS.md, Self-Improving Agents (ICLR '25)
Reasoned

Failures teach more than successes

The ExpeL framework found failure-derived heuristics substantially outperform success-derived ones. Anti-patterns that prune bad strategies are more valuable than positive guidance.

ExpeL extending Reflexion · cited in addyosmani.com/blog/self-improving-agents
Reasoned

The feedback loop is recursive

tactic_feedback() tracks which tactics get used and whether sessions improve. Activation rates calibrate promotion thresholds. Myelin generates meta-tactics about its own effectiveness. The evaluation harness IS the self-improvement mechanism.

04 The architecture decision

Three candidates. Five weighted criteria (correctness, effort, self-improvement, safety, ecosystem fit). Scored after peer review.

Auto-Promote

37
Fast but repeats the AGENTS.md failure mode

Commons + Human Gate

46
Evidence-backed · the human gate is proven

Standalone

31
Two daemons, two DBs · over-engineered

Core distillation lives in claude-session-commons. Myelin is the MCP server and CLI that exposes it — same architecture as resume-resume. All tactics require human approval to activate. Auto-promotion is a V2 path, earned after the feedback loop proves which tactic types are safe.

AGENTS.md is a notebook.
Myelin is a nervous system.

A notebook can't observe its own impact. It can't learn which instructions help and which hurt. It can't retire stale advice or promote proven patterns. Myelin can.
